## RackNerd 速度測試到底要測什麼

三件事，缺一不可。

**延遲（ping）**：數據包從你的電腦到伺服器再回來的往返時間，單位毫秒。延遲越低，SSH 操作、網頁響應越跟手。對於中國大陸用戶來說，美國機房的 ping 值一般在 150-250ms 之間，低於 180ms 就算不錯了。

**帶寬（下載/上傳速度）**：RackNerd 所有 VPS 套餐標配 1Gbps 端口，但 1Gbps 是端口上限，實際能跑多少取決於機房負載、線路質量、以及你本地的網絡環境。標稱 1Gbps 不代表你能跑滿 125MB/s。

**路由（traceroute/mtr）**：數據包從你到伺服器走了哪些節點。路由決定了穩定性和高峰期的表現——同樣的 ping 值，走優化線路的和走普通骨幹的，晚高峰體驗天差地別。

## 三網路由分析：電信、聯通、移動各走什麼線

路由才是決定晚高峰體驗的關鍵。同樣 ping 值 180ms，路由好的線路晚高峰照樣穩，路由差的線路一到晚上就丟包。

**電信**：RackNerd 大部分機房走的是 163 骨幹網直連 Cogent，路由比較乾淨，跳數不多。163 骨幹在高峰時段會有策略性丟包，這是電信出口帶寬有限導致的，跟 RackNerd 沒關係。聖何塞和洛杉磯 DC-02 的電信線路表現相對穩定。

**聯通**：聖何塞機房聯通走的是 AS4837 優化線路，這在 RackNerd 的機房裡算表現最好的。聯通用戶整體體驗通常比電信和移動好一截，延遲低、穩定性也不錯。

**移動**：移動線路是三網裡表現最一般的，延遲偏高，路由也不太理想。RackNerd 的移動線路基本走 CMI 出口，聖何塞機房移動延遲能到 190ms 左右，其他機房更高。如果你主力是移動網絡，建議優先考慮聖何塞或西雅圖。

一句話總結：聯通用戶選聖何塞最穩，電信用戶選洛杉磯 DC-02 或聖何塞，移動用戶——說實話 RackNerd 對移動不算特別友好，能接受 200ms 左右延遲的話可以選聖何塞或西雅圖。

## 自己動手測 RackNerd 速度：四步搞定

不用買了才測，下單之前就能把速度摸個底。

1. **ping 延遲測試**：打開終端（Windows 用 cmd，Mac 用 Terminal），輸入 `ping 204.13.154.3`（替換成你要測的機房 IP），看平均延遲和丟包率。建議不同時段各測一次，白天和晚高峰對比著看。

2. **traceroute 路由追蹤**：輸入 `traceroute 204.13.154.3`（Windows 用 `tracert`），看數據包經過了哪些節點。跳數越少越好，超過 20 跳說明路由繞了。重點看最後幾跳有沒有丟包。

3. **下載測速文件**：在 VPS 上或者本地用 wget/curl 下載對應機房的測速文件，比如 `wget https://lg-lax02.racknerd.com/test_files/1GB.test`，看實際下載速度。1GB 的文件最能反映真實帶寬。

4. **用 Looking Glass 跑完整測試**：不想敲命令的話，直接打開對應機房的 Looking Glass 頁面（上面表格裡有地址），頁面上能直接跑 ping、traceroute 和下載測試，一鍵搞定。
